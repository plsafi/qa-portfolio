## Learning Notes (post testing conclusions)

1. Noticed that critical issues often appear in core user flows like cart and checkout
2. Found multiple bugs related to missing validation, which suggests weak backend checks
3. Realized the importance of testing edge cases (e.g. quantity = 0)
4. Observed that UI issues can significantly impact usability, even if functionality works
5. Detected inconsistencies between frontend display and actual system behavior
6. Learned to verify calculations both on item level and total level
7. Cross-browser testing revealed layout differences not visible in a single browser
8. Some errors (e.g. 404, unauthorized) indicate missing or misconfigured backend components
9. Reinforced the need to test error messages and system feedback
10. Identified importance of checking data mapping in user profile and invoices
